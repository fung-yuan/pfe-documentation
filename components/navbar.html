Navbar Overview

The Navbar component serves as the top navigation bar in the ESTLM application. It provides access to the theme toggle functionality and notifications.

The Navbar is a simple component that focuses on providing quick access to application-wide utilities rather than page-specific navigation (which is handled by the Sidebar).

Implementation

The Navbar component is implemented in client/src/components/layout/Navbar.jsx. It's a stateless functional component that renders a header with action buttons. 

import React from 'react';
import { Button } from "@/components/ui/button";
import { Bell } from "lucide-react";
import { ThemeToggle } from './ThemeToggle';

export function Navbar() {
  return (
    <header className="bg-card text-card-foreground border-b sticky top-0 z-10">
      <div className="container mx-auto h-16 px-4 flex items-center justify-end">
        <div className="flex items-center gap-2">
          <ThemeToggle />
          <Button variant="ghost" size="icon" aria-label="Notifications">
            <Bell className="h-5 w-5" />
          </Button>
        </div>
      </div>
    </header>
  );
}

Styling and Layout

The Navbar uses the following key styling features:

  • Fixed Height (h-16): Consistent 64px height
  • Sticky Positioning: Remains visible at the top of the viewport when scrolling
  • Z-Index (z-10): Ensures it stays above other content
  • Border Bottom: Provides visual separation from content
  • Right-Aligned Content: All navbar items are aligned to the right

ThemeToggle Component

The Navbar includes the ThemeToggle component, which allows users to switch between light and dark themes. This component leverages the application's theme context to toggle between different visual modes.

The ThemeToggle component is imported from ./ThemeToggle and is responsible for:

  • Detecting the current theme (light or dark)
  • Providing a toggle button to switch between themes
  • Updating the theme in both the UI and localStorage for persistence

Usage

The Navbar component is used in the Layout component to provide the top navigation bar for the entire application: 

// From Layout.jsx
import React from 'react';
import { Sidebar } from './Sidebar';
import { Navbar } from './Navbar';

function Layout({ children }) {
  return (
    <div className="flex h-screen overflow-hidden bg-background">
      <Sidebar />
      <div className="flex flex-col flex-1 overflow-hidden">
        <Navbar />
        <main className="flex-1 overflow-auto p-4">
          {children}
        </main>
      </div>
    </div>
  );
}

The Navbar doesn't require any props as it's a self-contained component that:

  • Renders consistently across all authenticated pages
  • Accesses the theme context directly for theme toggling
  • Doesn't need page-specific customization

Related Components

The Navbar component works with these other components:

  • Layout - The parent component that renders the Navbar
  • ThemeToggle - Used within the Navbar for theme switching
  • Sidebar - Works alongside the Navbar to provide navigation
