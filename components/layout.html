Layout Overview

The Layout component serves as the main structural container for the ESTLM application. It provides a consistent framework for all authenticated pages, wrapping the content with the navigation sidebar and top navbar.

This component is used in AppRoutes.jsx to wrap all authenticated routes, ensuring they all share the same structure and navigation elements.

Main Content Area

This area contains the page-specific content rendered via {children}

Implementation

The Layout component is implemented in client/src/components/layout/Layout.jsx. It uses a flexbox-based layout to create a responsive structure with a fixed sidebar and a scrollable content area. 

import React from 'react';
import { Sidebar } from './Sidebar';
import { Navbar } from './Navbar';

function Layout({ children }) {
  return (
    <div className="flex h-screen overflow-hidden bg-background">
      <Sidebar />
      <div className="flex flex-col flex-1 overflow-hidden">
        <Navbar />
        <main className="flex-1 overflow-auto p-4">
          {children}
        </main>
      </div>
    </div>
  );
}

export default Layout;

Component Structure

The Layout component consists of:

  • Root Container: A flex container that takes the full height of the screen with overflow hidden
  • Sidebar: The fixed-width navigation sidebar on the left
  • Main Content Area: A flex column container with:
    • Navbar: The top navigation bar
    • Main Content: The scrollable area where page content is rendered

Props

Prop Type Required Description
children React Node Yes The page content to be rendered in the main content area

Usage

The Layout component is used in AppRoutes.jsx to wrap all authenticated routes: 

// From AppRoutes.jsx
<Route 
  path="/*"
  element={
    isAuthenticated ? (
      <Layout>
        <Routes>
          <Route path="/" element={<Navigate to="/dashboard" replace />} />
          <Route path="dashboard" element={<Dashboard />} />
          <Route path="students" element={<StudentManagement />} />
          {/* Other routes... */}
        </Routes>
      </Layout>
    ) : (
      <Navigate to="/login" replace />
    )
  }
/>

This usage pattern ensures that:

  • All authenticated pages have a consistent layout
  • Navigation between pages doesn't require re-rendering the entire layout
  • Only the inner content changes when navigating between routes
Note: The Layout is only applied to authenticated routes. The login page has its own separate layout without the sidebar and navbar.
