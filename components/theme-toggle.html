Component Overview

The ThemeToggle component provides a simple and elegant way for users to switch between light and dark themes in the ESTLM application. It uses the next-themes library to manage theme state and persistence.

The theme toggle is displayed in the Navbar component and allows users to switch between light and dark themes with a single click. The selected theme is persisted across page reloads and browser sessions.

Implementation

The ThemeToggle component is implemented in client/src/components/layout/ThemeToggle.jsx as a functional React component. It uses the useTheme hook from the next-themes library to interact with the application's theme context. 

import React from 'react';
import { Button } from "@/components/ui/button";
import { useTheme } from "next-themes"; 
import { Sun, Moon } from "lucide-react";

export function ThemeToggle() {
  const { theme, setTheme } = useTheme();

  const currentTheme = theme === 'system' 
    ? (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light') 
    : theme;

  const toggleTheme = () => {
    setTheme(currentTheme === 'dark' ? 'light' : 'dark');
  };

  return (
    <Button
      variant="ghost"
      size="icon"
      onClick={toggleTheme}
      aria-label="Toggle theme"
    >
      {currentTheme === 'dark' ? (
        <Sun className="h-[1.2rem] w-[1.2rem]" />
      ) : (
        <Moon className="h-[1.2rem] w-[1.2rem]" />
      )}
    </Button>
  );
}

How It Works

The ThemeToggle component functions through the following steps:

  1. Theme Detection: On initialization, it checks if a theme has been previously set (theme value from useTheme). If not, it falls back to the system's preferred color scheme using the window.matchMedia API.
  2. Toggle Function: When clicked, the component calls toggleTheme() which switches between 'light' and 'dark' themes by calling the setTheme function from the next-themes library.
  3. Visual Feedback: The component displays either a sun or moon icon depending on the current theme, providing clear visual indication of the active theme.
  4. Theme Persistence: The next-themes library handles persisting the theme preference to localStorage, ensuring the user's preference is remembered across page reloads.

Integration with the Theme System

This component integrates with a broader theme system that:

  • Applies CSS classes to the <html> element based on the selected theme
  • Works with Tailwind CSS's dark mode which uses class strategy
  • Persists theme selection in the browser's localStorage
  • Provides context through ThemeProvider for theme-aware components

Usage

The ThemeToggle component is used in the Navbar component to provide the theme switching functionality: 

// From Navbar.jsx
import { ThemeToggle } from './ThemeToggle';

export function Navbar() {
  return (
    <header className="bg-card text-card-foreground border-b sticky top-0 z-10">
      <div className="container mx-auto h-16 px-4 flex items-center justify-end">
        <div className="flex items-center gap-2">
          <ThemeToggle />
          <Button variant="ghost" size="icon" aria-label="Notifications">
            <Bell className="h-5 w-5" />
          </Button>
        </div>
      </div>
    </header>
  );
}

The component is designed to be self-contained and doesn't require any props to function, making it easy to integrate anywhere in the application.

Dependencies

For this component to work correctly, the application needs:

  • The next-themes library installed
  • A ThemeProvider wrapping the application
  • Proper Tailwind CSS configuration for dark mode using the 'class' strategy

Accessibility Considerations

The ThemeToggle component incorporates several accessibility features:

  • Uses the aria-label attribute to provide a descriptive label for screen readers
  • Relies on the Button component which includes proper keyboard navigation support
  • Provides clear visual indication of the current theme via the sun/moon icons
  • Maintains sufficient contrast in both light and dark modes

These considerations ensure that all users, regardless of ability, can effectively use the theme toggle feature.
