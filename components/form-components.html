Overview

The ESTLM Application uses a comprehensive set of form components that provide a consistent, accessible, and user-friendly way to collect and validate user input. These components are built on top of radix-ui primitives and react-hook-form, providing a powerful foundation for form implementation.

The form components in the application are designed with the following principles:

  • Consistency - Unified styling and behavior across all input types
  • Accessibility - Proper ARIA attributes and keyboard navigation
  • Validation - Built-in support for form validation
  • Flexibility - Components that adapt to different data types and requirements
  • Internationalization - Support for both English and French input

Input Component

The Input component is a styled wrapper around the HTML <input> element, providing consistent styling and behavior across the application. 

// From input.tsx
import * as React from "react"
import { cn } from "@/lib/utils"

const Input = React.forwardRef>(
  ({ className, type, ...props }, ref) => {
    return (
      <input
        type={type}
        className={cn(
          "flex h-9 w-full rounded-md border border-input bg-transparent px-3 py-1 text-base shadow-sm transition-colors file:border-0 file:bg-transparent file:text-sm file:font-medium file:text-foreground placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-ring disabled:cursor-not-allowed disabled:opacity-50 md:text-sm",
          className
        )}
        ref={ref}
        {...props}
      />
    )
  }
)
Input.displayName = "Input"

export { Input }

Key Features

  • Consistent Styling - Maintains the application's design language
  • Type Support - Works with all HTML input types (text, email, password, etc.)
  • Responsive Design - Adapts to different screen sizes
  • Focus States - Clear visual indication when focused
  • Disabled States - Proper styling for disabled inputs

Usage Example

<Input
  type="text"
  placeholder="Enter student name"
  value={studentName}
  onChange={(e) => setStudentName(e.target.value)}
  className="w-full"
/>

Select Component

The Select component provides a dropdown selection interface, built on top of Radix UI's select primitive for accessibility and keyboard navigation.

Key Components

  • Select - Root component that manages selection state
  • SelectValue - Displays the currently selected value
  • SelectTrigger - The button that opens the dropdown
  • SelectContent - Contains the list of selectable options
  • SelectItem - Individual selectable option

Usage Example

<Select value={department} onValueChange={setDepartment}>
  <SelectTrigger className="w-full">
    <SelectValue placeholder="Select Department" />
  </SelectTrigger>
  <SelectContent>
    {departments.map(dept => (
      <SelectItem key={dept.id} value={dept.id.toString()}>
        {dept.name}
      </SelectItem>
    ))}
  </SelectContent>
</Select>

Internationalization Support

The Select component fully supports displaying and selecting options with French accented characters, which is important for department names and other multilingual content in the application.

DatePicker Component

The DatePicker component provides an intuitive calendar interface for selecting dates, built using the Popover and Calendar components. 

// From DatePicker.tsx
export function DatePicker({
  date,
  setDate,
  label,
  placeholder = "Pick a date",
  className
}: DatePickerProps) {
  return (
    <div className={cn("space-y-2", className)}>
      {label && <div className="text-sm font-medium text-foreground mb-1">{label}</div>}
      <Popover>
        <PopoverTrigger asChild>
          <Button
            variant={"outline"}
            className={cn(
              "w-full justify-start text-left font-normal",
              !date && "text-muted-foreground"
            )}
          >
            <CalendarIcon className="mr-2 h-4 w-4" />
            {date ? format(date, "PPP") : <span>{placeholder}</span>}
          </Button>
        </PopoverTrigger>
        <PopoverContent className="w-auto p-0">
          <Calendar
            mode="single"
            selected={date}
            onSelect={setDate}
            initialFocus
          />
        </PopoverContent>
      </Popover>
    </div>
  )
}

Key Features

  • Calendar Interface - Intuitive calendar popup for date selection
  • Date Formatting - Properly formatted date display using date-fns
  • Internationalization - Support for different date formats
  • Keyboard Navigation - Accessible date selection via keyboard
  • Custom Labeling - Optional label for better form organization

Usage in the Attendance Record Form

The DatePicker is a critical component in the Attendance Record Form, allowing instructors to select the date for a class session when recording student attendance. 

<DatePicker 
  label="Date"
  date={date}
  setDate={setDate}
  placeholder="Select attendance date"
/>

Form Components

The Form components provide a structured way to create forms with validation and error handling, built on top of react-hook-form for state management and validation.

Key Components

  • Form - Provides form context using react-hook-form's FormProvider
  • FormField - Connects individual form fields to the form context
  • FormItem - Wrapper for a form input and its associated elements
  • FormLabel - Accessible label for form inputs
  • FormControl - Connects inputs to the form field context
  • FormDescription - Provides help text for form fields
  • FormMessage - Displays validation error messages

Example Form Implementation

// Form with validation example
const form = useForm({
  resolver: zodResolver(formSchema),
  defaultValues: {
    studentName: "",
    departmentId: "",
    attendanceDate: new Date(),
  },
});

return (
  <Form {...form}>
    <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-8">
      <FormField
        control={form.control}
        name="studentName"
        render={({ field }) => (
          <FormItem>
            <FormLabel>Student Name</FormLabel>
            <FormControl>
              <Input placeholder="Student name" {...field} />
            </FormControl>
            <FormDescription>
              Enter the student's full name
            </FormDescription>
            <FormMessage />
          </FormItem>
        )}
      />
      
      {/* Additional form fields... */}
      
      <Button type="submit">Submit</Button>
    </form>
  </Form>
)

Integration with Absence Management

The form components are essential to the absence management system, enabling data collection for:

  • Attendance Recording - Capturing student presence/absence data
  • Department Selection - Filtering students by department
  • Date Selection - Specifying the class session date
  • Search Parameters - Inputting natural language search queries

These components support French accented characters and special characters, which is critical for the natural language search functionality that allows querying student absence data in both English and French.

Internationalization Support

Form components in the ESTLM application are designed to work seamlessly with:

  • French accented characters (√©, √®, √™, √ß, etc.)
  • Different date formats (European vs. North American)
  • Multilingual content in department and student names
  • Natural language input in both English and French
