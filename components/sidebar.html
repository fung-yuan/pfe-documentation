Sidebar Overview

The Sidebar component provides the main navigation for the ESTLM application. It includes links to all main sections of the application and can be collapsed to save space on smaller screens.

The sidebar's collapsed state is persisted in localStorage, so it remembers the user's preference across sessions.

Implementation

The Sidebar component is implemented in client/src/components/layout/Sidebar.jsx. It uses React's useState and useEffect hooks to manage the collapsed state and persist it to localStorage.

Key Features

  • Collapsible Interface: Can be toggled between expanded and collapsed states
  • State Persistence: Remembers the collapsed state in localStorage
  • Active Route Highlighting: Uses React Router's NavLink to highlight the active route
  • Tooltips in Collapsed Mode: Shows tooltips when hovering over icons in collapsed mode
  • Lucide React Icons: Uses consistent icons from the Lucide React library
  • Quick Access Footer: Provides quick access to settings and logout

State Management

The sidebar uses React's useState hook to track whether it's collapsed or expanded: 

// From Sidebar.jsx - State management for collapsed state
const [collapsed, setCollapsed] = useState(() => {
  const savedState = localStorage.getItem('sidebarCollapsed');
  return savedState ? JSON.parse(savedState) : false;
});

// Save collapsed state to localStorage whenever it changes
useEffect(() => {
  localStorage.setItem('sidebarCollapsed', JSON.stringify(collapsed));
}, [collapsed]);

NavLink Integration

The sidebar uses React Router's NavLink component to create navigation links with active state styling: 

{routes.map((route) => {
  const IconComponent = Icons[route.icon] || Icons.Package;
  return (
    <NavLink
      key={route.path}
      to={route.path}
      title={collapsed ? route.name : undefined}
      className={({ isActive }) => cn(
        "flex items-center p-2 rounded-md hover:bg-accent hover:text-accent-foreground transition-all duration-200",
        isActive ? "bg-accent/20 text-primary font-medium border-l-4 border-primary pl-[10px]" : "text-muted-foreground hover:text-foreground",
        collapsed ? "justify-center" : "justify-start",
        isActive && collapsed && "border-l-4 border-primary"
      )}
    >
      {IconComponent && <IconComponent className="w-5 h-5" />}
      {!collapsed && <span className="ml-2 whitespace-nowrap">{route.name}</span>}
    </NavLink>
  );
})}

This implementation:

  • Maps over the centralized routes configuration to create navigation links
  • Uses the isActive state from NavLink to style active routes
  • Conditionally renders text labels based on the collapsed state
  • Adds tooltips (via title attribute) when in collapsed mode
  • Uses Lucide React icons specified in the route configuration

Usage

The Sidebar component is used in the Layout component to provide navigation for the entire application: 

// From Layout.jsx
import React from 'react';
import { Sidebar } from './Sidebar';
import { Navbar } from './Navbar';

function Layout({ children }) {
  return (
    <div className="flex h-screen overflow-hidden bg-background">
      <Sidebar />
      <div className="flex flex-col flex-1 overflow-hidden">
        <Navbar />
        <main className="flex-1 overflow-auto p-4">
          {children}
        </main>
      </div>
    </div>
  );
}

The Sidebar doesn't require any props as it gets all its required data from:

  • The centralized routes configuration in routes.jsx
  • The authentication context via useAuth() hook for the logout functionality
  • localStorage for persisting the collapsed state

Related Components and Files

The Sidebar component works with these other components and files:

  • Layout - The parent component that renders the Sidebar
  • routes.jsx - The centralized route configuration used to generate navigation links
  • context/AuthContext.jsx - Provides the logout function for the sidebar's logout button
