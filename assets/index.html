Assets Overview

The Absence Management System manages various static assets including images, icons, fonts, and other media files. These assets are organized in a structured way to maintain consistency and optimize performance.

This documentation outlines the asset organization, usage guidelines, and best practices for maintaining and extending the asset library.

Asset Usage in Code

Assets are imported and used in components following React's asset import patterns: 

// Importing image assets
import logo from '@/assets/images/logo.svg';
import placeholderUser from '@/assets/images/placeholder-user.png';

// Usage in component
const Header = () => {
  return (
    <header>
      <img src={logo} alt="Absence Management System Logo" />
      {/* other header content */}
    </header>
  );
};

// Importing and using SVG icons
import { ReactComponent as DashboardIcon } from '@/assets/icons/dashboard.svg';

const Sidebar = () => {
  return (
    <nav>
      <ul>
        <li>
          <DashboardIcon className="icon" />
          <span>Dashboard</span>
        </li>
        {/* other nav items */}
      </ul>
    </nav>
  );
};

Asset Best Practices

  • Optimization: All images are optimized for web using appropriate formats:
    • SVG for icons and logos
    • WebP or optimized JPG/PNG for photos
    • WOFF2 for font files
  • Lazy Loading: Non-critical images are lazy-loaded to improve initial load performance: 
    <img src={image} loading="lazy" alt="Description" />
  • Responsive Images: Images are made responsive using appropriate CSS techniques: 
    img {
  max-width: 100%;
  height: auto;
}
  • Alt Text: All images include descriptive alt text for accessibility
  • Theme Support: Where appropriate, assets have light and dark versions to support theming
  • Vector Priority: SVG is preferred for icons and logos for crisp rendering at any size

Adding New Assets

When adding new assets to the application, follow these guidelines:

  1. Directory Structure: Place the asset in the appropriate directory based on its type
  2. Naming Conventions: Use kebab-case for file names (e.g., student-profile.jpg)
  3. Optimization: Optimize all assets before adding them to the repository
  4. Documentation: Update the asset documentation to include new assets
  5. Version Control: Commit the assets separately from code changes for clearer history
