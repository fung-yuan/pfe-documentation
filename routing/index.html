Understanding the Routing System

The Absence Management System uses React Router v6 to handle navigation and route management. This allows the application to function as a single-page application (SPA) where navigation happens without full page reloads, providing a smooth user experience.

The routing system also includes authentication protection for routes, redirects for unauthenticated users, and role-based access control for certain parts of the application.

Route Structure

The application's routes are defined in AppRoutes.jsx, which sets up the main routing structure and authentication protection. Here's a simplified view of the routing code: 

const AppRoutes = () => {
  const { isAuthenticated, userRole } = useAuth();

  return (
    <ErrorBoundary>
      <Routes>
        {/* Public login route */}
        <Route path="/login" element={<LoginPage />} />

        {/* Protected routes wrapped by Layout */}
        <Route 
          path="/*"
          element={
            isAuthenticated ? (
              <Layout>
                <Routes>
                  <Route path="/" element={<Navigate to="/dashboard" replace />} />
                  <Route path="dashboard" element={<Dashboard />} />
                  <Route path="send-email-announcement" element={
                    <ProtectedRoute isAllowed={userRole === 'ADMIN'}>
                      <SendEmailAnnouncementPage />
                    </ProtectedRoute>
                  } />
                  <Route path="students" element={<StudentManagement />} />
                  <Route path="attendance" element={<AttendanceManagement />} />
                  <Route path="reports" element={<Reports />} />
                  <Route path="departments" element={<DepartmentManagement />} />
                  <Route path="subjects" element={<SubjectManagement />} />
                  <Route path="search" element={<NaturalLanguageSearchPage />} />
                  <Route path="profile" element={<AdminProfilePage />} />
                  <Route path="*" element={<div>Page Not Found</div>} />
                </Routes>
              </Layout>
            ) : (
              <Navigate to="/login" replace />
            )
          }
        />
      </Routes>
    </ErrorBoundary>
  );
};

Authentication Flow

The routing system is tightly integrated with the authentication system. Here's how the authentication flow works with routing:

User Visits Page
→
Check Authentication
→
Authenticated? Yes/No
→
Render Route or Redirect

If a user attempts to access a protected route without being authenticated, they are automatically redirected to the login page. After successful authentication, they are redirected back to the original route they were trying to access.

Available Routes

The application includes the following routes:

Path Component Access Description
/login LoginPage Public User authentication page
/ Redirect to Dashboard Authenticated Root path redirects to dashboard
/dashboard Dashboard Authenticated Main dashboard with overview statistics
/students StudentManagement Authenticated Student management interface
/attendance AttendanceManagement Authenticated Attendance tracking and management
/reports Reports Authenticated Reporting and analytics
/departments DepartmentManagement Authenticated Department management
/subjects SubjectManagement Authenticated Subject/course management
/search NaturalLanguageSearchPage Authenticated AI-powered search interface
/send-email-announcement SendEmailAnnouncementPage Admin only Send emails and announcements
/profile AdminProfilePage Authenticated User profile management
