Natural Language Search

Advanced search capabilities for student absence management using natural language processing

Overview

The Natural Language Search feature enables users to find student absence information using everyday language instead of complex database queries. Users can simply type questions or requests in a conversational manner, and the system will interpret their intent and return relevant results.

Unlike traditional search systems that rely on exact keyword matching or specific syntax, our natural language search capabilities allow administrators, teachers, and staff to interact with the attendance data in an intuitive, human-like way.

Recent Improvements

Our natural language search has been significantly enhanced with several key improvements:

  • GPT-4 Upgrade: Upgraded from GPT-3.5-turbo to GPT-4 for more accurate and nuanced query generation.
  • Zero Absence Handling: Implemented special case handling for queries about perfect attendance using a direct "NOT EXISTS" database approach for accurate identification of students with zero absences.
  • Absence Calculation Fix: Improved absence hour calculations with validation and capping of unreasonable values.
  • Enhanced Summaries: Better natural language summaries of search results for faster understanding of query outcomes.
  • Language Support: Improved handling of French accented characters in search queries.
  • Schema Context: Added more detailed database schema information to the AI prompt for more accurate query generation.
  • Improved Debugging: Added better logging throughout the system for easier troubleshooting.

How It Works

The natural language search feature uses a sophisticated process to convert user queries into database operations:

1
User Query

The user enters a natural language query such as "Show me students with zero absences" or "Find students in the Computer Science department who missed more than 5 classes."

2
Query Processing

The frontend sends the query to the backend service, which prepares it for AI processing by sanitizing inputs and adding context.

3
GPT-4 Analysis

The OpenAI GPT-4 model analyzes the query, understanding the user's intent and identifying key parameters (like department names, date ranges, or absence thresholds).

4
JPQL Generation

Based on the AI analysis, the system generates an optimized JPQL (Java Persistence Query Language) query with appropriate filters, joins, and conditions.

Special case handling applies for specific query types like zero absence queries, which use a "NOT EXISTS" subquery approach.

5
Database Execution

The generated JPQL query is executed against the database to retrieve matching student records and their absence data.

6
Result Processing

The system processes the database results, calculating aggregate statistics and preparing the data for display.

7
Summary Generation

The system generates a natural language summary of the results to help users quickly understand what was found.

8
User Display

The frontend displays the results to the user, including the summary, student list, and relevant absence details.

Special Case Handling

For queries about students with zero absences, we use a specialized approach with a "NOT EXISTS" subquery pattern rather than counting absences. This ensures accurate results even when there are no absence records at all for a student.

Implementation

The Natural Language Search feature is implemented through a combination of frontend and backend components:

Backend: NaturalLanguageQueryService

NaturalLanguageQueryService.java
@Service
public class NaturalLanguageQueryService {
    private final OpenAIClient openAIClient;
    private final DatabaseSchemaService schemaService;
    private final Logger logger = LoggerFactory.getLogger(NaturalLanguageQueryService.class);
    
    // OpenAI configuration
    private static final String GPT_MODEL = "gpt-4"; // Upgraded from gpt-3.5-turbo
    
    @Autowired
    public NaturalLanguageQueryService(OpenAIClient openAIClient, DatabaseSchemaService schemaService) {
        this.openAIClient = openAIClient;
        this.schemaService = schemaService;
    }
    
    public List<StudentAbsenceDTO> processNaturalLanguageQuery(String query) {
        // Log the original query for debugging
        logger.info("Processing natural language query: {}", query);
        
        try {
            // Get database schema information to provide context to the AI
            String schemaInfo = schemaService.getDatabaseSchemaDescription();
            
            // Generate JPQL query using GPT-4
            String jpqlQuery = generateJPQLQuery(query, schemaInfo);
            logger.info("Generated JPQL query: {}", jpqlQuery);
            
            // Handle special case for zero absence queries
            if (jpqlQuery.contains("HAVING COUNT(a) = 0") || 
                query.toLowerCase().contains("zero") || 
                query.toLowerCase().contains("no absence")) {
                logger.info("Detected zero absence query, using specialized approach");
                return handleZeroAbsenceQuery();
            }
            
            // Execute the query and return results
            List<StudentAbsenceDTO> results = executeJPQLQuery(jpqlQuery);
            logger.info("Query returned {} results", results.size());
            return results;
        } catch (Exception e) {
            logger.error("Error processing natural language query", e);
            throw new QueryProcessingException("Failed to process query: " + e.getMessage());
        }
    }
    
    private List<StudentAbsenceDTO> handleZeroAbsenceQuery() {
        // Use NOT EXISTS approach for accurate zero absence detection
        String optimizedQuery = "SELECT s FROM Student s WHERE NOT EXISTS " +
                              "(SELECT a FROM Attendance a WHERE a.student = s)";
        return executeJPQLQuery(optimizedQuery);
    }
    
    private String generateJPQLQuery(String naturalQuery, String schemaInfo) {
        // Prepare prompt with schema context and query instructions
        String prompt = "Database Schema Information:\n" + schemaInfo + "\n\n" +
                     "Generate a JPQL query to find student absence information based on this natural language query: \n" +
                     "\"" + naturalQuery + "\"\n\n" +
                     "Return only the JPQL query without any explanation.";
        
        // Call OpenAI API with GPT-4 model
        return openAIClient.generateText(prompt, GPT_MODEL);
    }
}

Frontend: SearchComponent

SearchComponent.jsx
import React, { useState } from 'react';
import { Input, Button, Spin, List, Typography, Alert } from 'antd';
import { SearchOutlined } from '@ant-design/icons';
import SearchService from '@/services/SearchService';

const { Title, Text } = Typography;

const SearchComponent = () => {
    const [query, setQuery] = useState('');
    const [loading, setLoading] = useState(false);
    const [results, setResults] = useState(null);
    const [error, setError] = useState(null);
    
    const handleSearch = async () => {
        if (!query.trim()) return;
        
        try {
            setLoading(true);
            setError(null);
            const data = await SearchService.naturalLanguageSearch(query);
            setResults(data);
        } catch (err) {
            console.error('Search error:', err);
            setError('Failed to process your search query. Please try again.');
        } finally {
            setLoading(false);
        }
    };
    
    const handleExampleClick = (exampleQuery) => {
        setQuery(exampleQuery);
    };
    
    const examples = SearchService.getExampleQueries();
    
    return (
        <div className="search-container">
            <Title level={3}>Natural Language Search</Title>
            <Text type="secondary">
                Search for student absences using natural language. Try questions like "Show students with zero absences"
                or "Find students in Computer Science with more than 5 absences."
            </Text>
            
            <div className="search-box">
                <Input.Search
                    placeholder="Enter your search query..."
                    value={query}
                    onChange={(e) => setQuery(e.target.value)}
                    onSearch={handleSearch}
                    enterButton={<Button type="primary" icon={<SearchOutlined />}>Search</Button>}
                    size="large"
                    loading={loading}
                    disabled={loading}
                />
            </div>
            
            <div className="example-queries">
                <Text strong>Example queries:</Text>
                <div className="example-tags">
                    {examples.map((example, index) => (
                        <Button 
                            key={index} 
                            size="small" 
                            type="dashed"
                            onClick={() => handleExampleClick(example)}
                        >
                            {example}
                        </Button>
                    ))}
                </div>
            </div>
            
            {error && <Alert message={error} type="error" showIcon />}
            
            {loading && <div className="loading-container"><Spin size="large" /></div>}
            
            {results && !loading && (
                <div className="results-container">
                    <div className="results-summary">
                        <Alert
                            message="Search Results"
                            description={results.summary || `Found ${results.students.length} students matching your query.`}
                            type="info"
                            showIcon
                        />
                    </div>
                    
                    <List
                        dataSource={results.students}
                        renderItem={student => (
                            <List.Item>
                                <List.Item.Meta
                                    title={`${student.firstName} ${student.lastName}`}
                                    description={`Department: ${student.department.name} | Total Absences: ${student.totalAbsenceHours} hours`}
                                />
                            </List.Item>
                        )}
                        locale={{ emptyText: 'No students found matching your criteria' }}
                    />
                </div>
            )}
        </div>
    );
};

Key Technical Improvements

1. Zero Absence Optimization

A significant improvement has been the handling of zero absence queries. Previously, the system struggled with identifying students who had perfect attendance (zero absences) because:

  • Traditional counting approaches couldn't distinguish between "no absences recorded" and "zero as the count of absences"
  • Simple JOIN-based queries would exclude students who had no absence records at all

Our solution uses a direct "NOT EXISTS" approach in JPQL:

Optimized Zero Absence Query
SELECT s FROM Student s WHERE NOT EXISTS 
(SELECT a FROM Attendance a WHERE a.student = s)

This approach correctly identifies all students with no absence records, ensuring accurate reporting of perfect attendance.

2. GPT-4 Integration

Upgrading from GPT-3.5-turbo to GPT-4 has significantly improved query understanding and accuracy. GPT-4 provides:

  • Better understanding of complex queries with multiple conditions
  • More accurate JPQL query generation with proper joins and conditions
  • Better handling of ambiguous language and intent
  • Improved recognition of domain-specific terms in education and attendance tracking

3. Schema-Informed Prompting

We've enhanced the AI prompts by including detailed database schema information, which helps the model generate more accurate queries by understanding:

  • The exact entity relationships between students, departments, and attendance records
  • Available fields and their data types for more precise filtering
  • The database structure to create correctly formatted JPQL queries

Best Practices for Using Natural Language Search

To get the most out of the Natural Language Search feature, follow these guidelines:

  1. Be specific but natural - Include key details like departments, time periods, or absence thresholds, but phrase your query as you would normally speak.
  2. Include clear metrics - Specify what you're measuring ("hours of absence" vs. "number of absences").
  3. Use date formats consistently - Standard date formats like YYYY-MM-DD work best.
  4. Experiment with examples - Try the example queries to understand the system's capabilities.
  5. Check result summaries - The system provides a summary of what it understood from your query.
